## 1. 背景与目标

将本地目录 `/Users/baowenzhuo/project/xhxagentv3/docs/bwz` 中的 Markdown、HTML、图片和附件发布为公开的 Project Radar 文档网站。本机文件变化后，网站应在数秒内更新；Mac 重新登录后，监听器应自动恢复运行。

设计目标如下：

- 网站公开只读，不要求访问者登录。
- 同步写入能力仅授予持有密钥的本机监听器。
- 普通文档变化不触发整站重新部署。
- 新增、修改、重命名和删除均能同步。
- 云端永远向读者暴露完整版本，避免展示同步到一半的内容。
- 网络中断、进程退出或重复保存不会导致数据损坏或永久漏同步。

## 2. 范围

### 2.1 首版包含

- 以 `README.md` 为首页的公开文档站。
- Markdown 渲染、原生 HTML 隔离展示、图片和附件访问。
- 桌面端分组侧边导航与移动端折叠导航。
- 稳定的文档深层链接和站内相对链接转换。
- R2 云端对象存储。
- 受同步密钥保护的上传与提交接口。
- macOS 文件监听、离线重试、登录自启动和日志。
- 构建、单元、接口、监听器与部署后验证。

### 2.2 首版不包含

- 全文搜索、评论、在线编辑和访问者账户。
- 文档版本浏览与回滚界面。
- 多个本地来源目录或多写入端并行同步。
- Git 托管和 CI 发布链路。
- 每次内容变化重新部署 Sites。

### 2.3 仓库隔离约束

站点项目与 AgentV3 源码仓库必须完全隔离：

- 站点设计文档、源码、测试、Sites 配置、同步器源码和相关 Git 提交只存在于 `/Users/baowenzhuo/Documents/Codex/2026-07-10/sites-plugin-sites-openai-bundled-2` 的独立 Git 仓库。
- `/Users/baowenzhuo/project/xhxagentv3` 仅作为只读内容来源；实现过程不得在其中创建站点文件、生成物、配置、分支或提交。
- 不修改 AgentV3 仓库的 `.gitignore`、索引、工作树或既有未提交改动。
- 本机监听器只读取 `/Users/baowenzhuo/project/xhxagentv3/docs/bwz` 中的内容，不在该目录写入状态文件、缓存、日志或锁文件，也不执行任何 Git 命令。
- 监听器运行文件、密钥配置和日志位于独立的 Application Support 目录；它们不进入 AgentV3 仓库。

站点采用 Sites 的动态服务端能力和一个 R2 逻辑绑定。公开读路径只读取 R2；私有写路径校验 `DOCS_SYNC_TOKEN` 后才允许上传或提交。同步密钥由 Sites 环境变量保存，本机副本放入权限为 `0600` 的配置文件，不写入源码、网页或 LaunchAgent 配置正文。

## 4. 组件与职责

### 4.1 公开文档站

- `/` 将 `README.md` 渲染为文档总览。
- `/docs/{document-path}` 根据路径打开相应 `.md`、`.html` 或 `.htm` 文档。
- 左侧导航按“核心文档”和“决策记录”分组；顺序优先沿用 `README.md` 中的推荐阅读顺序，其余项目按路径稳定排序。
- 移动端将导航折叠为可通过键盘和触摸操作的抽屉。
- 缺失文档返回明确的 404 页面，不回退到首页。

### 4.2 内容渲染器

- Markdown 支持标题、表格、代码块、引用、任务清单和普通链接。
- Markdown 中的原始 HTML 不执行；危险协议、事件属性和脚本内容被拒绝。
- 相对 `.md`、`.html`、`.htm` 链接转换为站内文档路由。
- 相对图片和附件链接转换为当前清单对应的内容哈希地址。
- 原生 HTML 通过受限 `iframe` 展示，不授予 `allow-same-origin` 或脚本执行权限。
- 原生 HTML 响应使用限制性 CSP：脚本禁用，只允许本站、`data:` 和明确允许的图片/样式来源。

### 4.3 R2 内容存储

对象采用内容寻址，键格式为 `objects/{sha256}`。当前清单存放在固定键 `manifests/current.json`，结构如下：

```json
{
  "schemaVersion": 1,
  "revision": "2026-07-10T12:00:00.000Z-a1b2c3d4",
  "generatedAt": "2026-07-10T12:00:00.000Z",
  "files": [
    {
      "path": "README.md",
      "sha256": "0123456789abcdef0123456789abcdef0123456789abcdef0123456789abcdef",
      "bytes": 1234,
      "mediaType": "text/markdown; charset=utf-8",
      "kind": "markdown"
    }
  ]
}
```

路径统一使用 UTF-8 和 `/` 分隔符，必须是来源目录内的相对路径。清单不包含隐藏文件、符号链接、目录项或绝对路径。

### 4.4 私有同步接口

同步接口提供三个操作：

1. 读取当前修订和已存在哈希，供监听器计算差异。
2. 上传缺失的内容对象；服务端重新计算 SHA-256 并拒绝哈希不匹配的请求。
3. 提交新清单；服务端确认清单中所有对象存在后，最后写入 `manifests/current.json`。

提交请求携带监听器读取到的 `baseRevision`。如果云端修订已经变化，服务端返回冲突，监听器重新拉取清单并再次计算差异。虽然首版只有一个写入端，这一校验可以防止旧任务覆盖新状态。

同步接口拒绝未授权请求、路径穿越、隐藏路径、符号链接描述、未知清单版本、无效哈希和超出限制的对象。首版限制为单文件不超过 20 MiB；超过限制的文件记录为同步错误且不会提交不完整清单。

### 4.5 本机监听器

监听器使用 Node.js 标准能力递归监听指定目录，不依赖编辑器插件。它负责：

- 启动时完整扫描并校验一次。
- 对连续文件事件执行约 1 秒防抖。
- 计算文件 SHA-256、大小、媒体类型和类别。
- 串行上传缺失对象并提交完整清单。
- 将同步过程中出现的新变化合并到下一轮。
- 失败时从约 2 秒开始指数退避，最长等待 5 分钟；任何新文件事件可提前唤醒重试。
- 网络恢复或发生修订冲突时重新进行完整差异计算。
- 记录时间、相对路径、结果和错误摘要，不记录密钥或文件正文。

所有普通非隐藏文件均可作为文档附件同步；`.md`、`.html` 和 `.htm` 可直接成为导航文档。符号链接始终拒绝，以免意外同步来源目录外的内容。

### 4.6 macOS 登录自启动

安装器将监听器运行副本和权限受限的配置放入 `~/Library/Application Support/ProjectRadarDocsSync/`，并在 `~/Library/LaunchAgents/` 安装用户级 LaunchAgent。LaunchAgent 在用户登录时启动监听器，并在监听器异常退出后重新启动。

LaunchAgent 配置只包含程序路径、工作目录和日志路径，不包含同步密钥。卸载脚本负责停止任务并删除 LaunchAgent；不会删除来源文档或云端内容。

## 5. 数据流与一致性

一次同步按以下顺序执行：

1. 扫描本地目录并构造候选清单。
2. 读取云端当前修订和已有对象哈希。
3. 上传云端缺失且本地仍存在的内容对象。
4. 再次确认本地扫描期间没有待处理变化；如有变化，重新扫描。
5. 使用 `baseRevision` 提交完整新清单。
6. 清单提交成功后，对不再被当前清单引用的旧对象执行清理。

公开站点只读取 `manifests/current.json`。因此上传中的对象对读者不可见；新清单提交前读者看到旧版本，提交后看到完整新版本。删除或重命名会先体现在新清单中，因此旧路径立即不再公开；清理中断只会暂时留下不可达对象，不影响公开内容，下一次同步会继续清理。

## 6. 页面与视觉体验

- 首页和正文采用浅色背景、深灰正文和低饱和蓝紫强调色。
- 版式优先保证中文长文、表格和代码块的阅读舒适度。
- 桌面端左侧固定导航，右侧为正文；正文宽度限制在适合阅读的范围。
- 移动端单栏展示，导航抽屉具备焦点管理、关闭按钮和无障碍标签。
- 页面展示当前文档标题、层级导航和最近一次云端同步时间。
- 内容哈希资源使用长期不可变缓存；清单和文档页面使用短缓存或重新验证，使更新在数秒内可见。

## 7. 安全设计

- 网站公开只读，同步接口必须提供 Bearer 密钥。
- 密钥使用常量时间比较；未授权响应不透露文件或清单是否存在。
- 写入接口限制请求方法、请求体大小、路径格式、媒体类型和哈希格式。
- Markdown 不执行内嵌 HTML 或 JavaScript。
- 原生 HTML 在无同源、无脚本权限的沙箱中运行，并配合 CSP。
- 相对路径解析后必须仍位于来源根目录；本机和服务端各验证一次。
- 日志、错误页和公开接口不返回密钥、绝对本机路径或文件正文。
- 同步密钥应使用高熵随机值，可通过更新 Sites 环境变量和本机配置进行轮换。

## 8. 故障处理

| 场景 | 处理方式 |
|---|---|
| 本机断网或 Sites 暂时不可用 | 保留最新待同步状态并自动退避重试 |
| 监听器异常退出 | LaunchAgent 自动重启，启动后执行全量校验 |
| 连续快速保存 | 防抖合并为一次同步 |
| 上传过程中再次修改文件 | 不提交旧候选清单，重新扫描后再提交 |
| 提交修订冲突 | 重新读取云端状态并重算差异 |
| 对象哈希不匹配 | 拒绝对象并重新读取本地文件 |
| 单文件超出 20 MiB | 记录明确错误，不提交不完整清单 |
| 当前清单暂时缺失 | 首页显示“内容尚未同步”，而非服务器错误详情 |
| 某文档在清单中但对象缺失 | 返回可诊断的 503，并记录对象哈希和路径摘要 |
