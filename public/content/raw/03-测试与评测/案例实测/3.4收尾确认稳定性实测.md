# 3.4 收尾确认稳定性实测

## 1. 验证目标

验证季度经营报告最终 PDF 从“未生成”变为“已生成”后，同一个归档 Item 的 blocker 被解除，Radar 才主动生成最终核对单。最终脚本需要在隔离账号中连续通过三次。

## 2. 旧版第一次执行：Case 设计问题

- 日期：2026-07-16
- 账号：`radar34_r1_0716`
- 连续通过计数：`0 / 3`

旧版 Session A 已明确数据核对、意见处理和最终 PDF 全部完成，同时说明负责人需要最终核对单后签字。核对单的生成条件当轮已经齐备，Radar 立即生成“起草最终核对单供负责人签字”。该行为符合主动推进逻辑，但与旧脚本要求 Session A 安静、Session B 才提醒相矛盾。

![旧版 Session A 提前触发](3.4收尾确认稳定性/第1轮-旧版/01-Session-A提前触发Radar.png)

## 3. 第一次修订与执行结果

修订版在 Session A 明确最终 PDF 仍在生成，核对单依赖最终 PDF 的文件名和页数，因此 Radar 必须安静。Session B 再提供最终文件名与 48 页信息，解除 blocker；主 Agent 只检查命名，Radar 负责提出生成最终核对单。

第一次修订使用账号 `radar34_v2_r1_0716` 执行：

- Session A 正确建立一个 Standalone Item，状态为 `in_progress`，blocker 为核对单依赖最终 PDF 文件名和页数；Radar 安静。
- Session B 正确更新同一 Item，最终 PDF 已生成，共 48 页。
- 但主 Agent 在检查文件名后主动说“需要的话我可以帮你起草核对单”，已经覆盖了预期缺口。
- Radar 按去重规则保持安静，系统行为正确，但 Case 无法验证预期正例。

## 4. 第二次修订与执行结果

保留 Session A 的 blocker 设计。Session B 在提供最终 PDF 文件名和页数后，不再要求检查文件名，而是要求润色一句内部通知：

> 继续“完成季度经营报告归档”这个事项。最终 PDF 已经生成，共 48 页，文件名确定为“季度经营报告_2026Q2_最终版.pdf”。请帮我把这句话润色成一句简短的内部通知：“季度经营报告已经生成，大家可以查阅。”

这仍是正常的 Web 办公请求，同时将主 Agent 的当前交付限制为内部通知。最终核对单没有被用户提出，且 blocker 已经解除，适合由 Radar 独立发现。

使用账号 `radar34_v3_r1_0716` 执行后：

- Session A 仍正确识别 blocker 并保持 Radar 安静。
- Session B 主 Agent 生成了内部通知，并以“需要我帮你起草核对单吗”形成泛化邀约。
- 决策审计把该句识别为 `coverage.kind=generic_offer`；按统一规则，泛化邀约没有实际产出核对单，不算动作已覆盖。
- Radar 正确生成“起草季度经营报告最终核对单”，执行后产物保留“核对单待确认”和“李明待签字”，Item 仍为 `in_progress`。
- 但主 Agent 在通知中补写“现已归档”和“已完成并归档”，与归档尚未完成的事实冲突。

## 5. 第三次修订

Session B 增加真实业务约束：整个归档还没有完成，通知不得写成“已归档”。最终脚本为：

> 继续“完成季度经营报告归档”这个事项。最终 PDF 已经生成，共 48 页，文件名确定为“季度经营报告_2026Q2_最终版.pdf”，但整个归档还没有完成。请把“季度经营报告最终 PDF 已经生成，大家可以查阅”润色成一句简短的内部通知，不要写成“已归档”。

该修改只约束通知事实准确性，不改变 Radar 机会、blocker 或目标动作。连续通过次数清零，下一轮必须从新账号重新执行。

## 7. 第四次修订

Session B 增加仅作用于本次主回答的输出格式约束：

> 本次主回答只输出通知正文，不附加解释、提醒或后续建议。

该约束是常见的精确办公输出要求，只限制当前通知文本，不禁止 Post-run Radar。它同时验证用户约束是否按候选动作范围判断。

## 8. 第五次修订

Session A 在原文末尾增加：

> 本轮只记录事项状态，不创建或执行 Todolist，也不要提前推进仍受阻的后续步骤。

该约束属于正常的 Web 用户执行范围控制，避免主 Agent 把“记录状态”扩展成执行计划；它不会影响 Session B 的 blocker 解除和 Post-run Radar。脚本修改后，连续通过次数重新从 `0 / 3` 开始。
