# 恢复简报三项目演示计划

**目标：** 在独立演示账号中准备三个跨自然日、尚未结束的 Project，验证恢复简报以 Project 为单位展示，并生成标准桌面截图。

**实现方式：** 不修改产品代码；仅通过现有 Project、Item、Event 和 Conversation State 接口写入可追溯的演示数据，再使用 Web 页面验证。

## 1. 案例设计

| Project | 项目状态 | Item 状态组合 | 一句话摘要目标 |
| --- | --- | --- | --- |
| 客户门户改版上线 | `in_progress` | 1 完成、1 进行中、1 未开始、1 阻塞 | 体现正常推进中存在局部阻塞 |
| 经营数据看板交付 | `waiting` | 1 完成、2 未开始、1 阻塞 | 体现关键数据权限未开通导致整体等待 |
| 季度经营复盘汇报 | `in_progress` | 2 完成、1 进行中、1 未开始 | 体现接近完成，只剩审核和最终稿 |

## 2. 执行步骤

- [ ] 在 `resume_demo_0717` 中幂等补齐后两个 Project、各自 Item、跨自然日 Event 和来源 Session。
- [ ] 直接读取 Resume Brief，确认返回三个 Project，且摘要只使用“已完成、进行中、尚未开始、被……阻塞”。
- [ ] 登录 Web 页面并刷新，确认默认正好展示三个 Project。
- [ ] 验证“查看项目”打开 Project Radar，左侧列出三个 Project。
- [ ] 以 `1920×1080` 截取恢复简报总览和项目详情，保存到 `docs/bwz/05-资源/图片/恢复简报三项目实测/`。

## 3. 验收标准

- 恢复简报以 Project 为卡片单位，每张卡片聚合其 Item 状态。
- 三个 Project 的标题、状态和摘要可以在一张桌面截图中完整辨认。
- Project Radar 左侧显示三个 Project，选中项目后可以查看对应 Item。
- 截图文件均为 `1920×1080`，右侧工作栏保持桌面布局，不下沉到页面底部。
- 数据准备与页面读取不改变现有 Project 的业务状态。
