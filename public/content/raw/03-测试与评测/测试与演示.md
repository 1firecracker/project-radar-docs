## 1. 验证目标与策略

V0 需要证明：

- 每个完整 QA 都能留下可追溯 Event，后台失败不会丢失记录。
- 不同 Session 的 Event 可以更新同一 Item 或 Project。
- 独立 Item、Project、Non Project 和待分类的边界清晰。
- 用户可以通过居中双栏弹窗查看和纠正自动整理结果。
- Organizer 能正确更新 Item 与 Project 的不同层级状态，代码守门不会把 Item 完成误当成 Project 完成。
- Radar 只在状态更新成功后生成，并且不干扰主对话。
- Radar 没有生成时能够区分产品静默、证据不足、同一动作已覆盖、来源或状态版本过期、非法输出和系统失败。
- 生成产物内部文字和泛化邀约不算主 Agent 直接覆盖；用户约束只作用于实际命中的候选动作。
- 用户接受后，AgentV3 产生真实结果，结果再次回流状态。

测试分层：

- **单元测试**：结构校验、`completion_scope`、Event 幂等、状态补丁、Project 守门、统计聚合、去重和 Context Bundle。
- **接口测试**：Overview、Event 时间线、纠正、权限、Radar 查询、接受和反馈。
- **集成测试**：Post-run 触发、Organizer、有限轮询、Re-entry 边界、隐藏上下文和 Radar 防递归。
- **人工端到端测试**：弹窗信息层级、来源 Session、低打扰体验和五分钟演示。

## 3. Demo 数据与准备

演示项目：

```text
Project：完成 Progress Agent V0
目标：完成状态整理、Radar 推送和可重复演示
必要 Item：
  1. 对齐产品与技术文档
  2. 实现 Project / Item / Event 数据层
  3. 实现 Radar 执行闭环
可选 Item：优化弹窗动画
```

准备两个来源 Session：

- Session A：讨论产品形态、Post-run 时机和 V0 范围。
- Session B：讨论 Project State、数据层级和完成标准。

演示前检查：

- [ ] 后端、前端、模型和数据库可用。
- [ ] 演示用户已登录，两个 Session 均可定位。
- [ ] 数据库不存在相同 `source_run_id` 和冲突的待处理 Radar。
- [ ] next questions 已关闭，避免与 Post-run 触发的 Organizer 并发使用同一 LLM。
- [ ] 准备一份预置 Project / Item / Event 与 Radar 作为失败兜底。
- [ ] 接受建议后可以真实生成目标产物。

## 4. 五分钟演示脚本

### 4.1 创建跨 Session 状态

在 Session A 完成一次关于 V0 产品形态的 QA。主 Agent 正常回答，Post-run 触发 Organizer 在后台保存 Event，并整理到“完成 Progress Agent V0”。

切换到 Session B，继续讨论 Project State。完成 QA 后，新 Event 更新同一 Project 下的“整理产品与技术文档”或“实现数据层”事项。

讲解重点：Session 是来源，Item / Project 才是长期状态。

### 4.2 查看并纠正

切换右侧 Radar Tab 查看当前建议，再点击“查看全部项目”打开居中双栏弹窗：左侧选择 Project，右侧展示 Item 进度和来自两个 Session 的 Event。

准备一条误入待分类的 Event，把它纠正到正确 Item；确认来源 Session 不变，相关状态重新计算。

讲解重点：自动整理可解释、可追溯、可纠正。

### 4.3 Radar 出现与执行

主回复结束后，如果状态存在关键缺口，输入框上方出现一条 Action Bar。用户查看依据，再点击主要动作。

AgentV3 根据隐藏 Context Bundle 生成计划、草稿、清单或总结；执行结果形成新 Event，原 Radar 进入 done，关联 Item 更新进展。

讲解重点：Radar 不是通知，而是基于持久状态发现并执行下一步。

### 4.4 完成与重新打开

用户明确表示某个 Item 已完成，弹窗中状态变为 completed。随后用一条明确反例演示重新打开，原完成记录仍保留在 Event 时间线。

讲解重点：系统追踪的不是聊天数量，而是一件事是否真正完成。

## 5. 兜底与最终验收

演示失败兜底：

- Organizer 失败时展示 Event 进入待分类，并切换到预置 Project / Item 数据。
- Radar 未生成时使用预置 Radar，但明确说明这是兜底数据。
- Agent 执行失败时展示 failed，并切换到预先保存的产出示例；V0 不提供已经进入 running 后的原 Radar 重跑。
- 网络或模型不可用时只能作为界面演示，不能宣称自动整理或真实执行成功。
