**目标：** 用户隔日返回 Web 端时，读取已有 Project、Item、Event，展示可收缩的多项目一句话恢复简报，并同时提供 Project 详情和来源 Session 两个入口。

**架构：** 新增只读 `Resume Brief` 服务与 API，不新增数据库表，也不调用 Organizer 或 Radar。前端新增独立恢复简报组件，复用现有 `ProjectRadarModal` 和 Session 选择逻辑完成双入口跳转。

**技术栈：** FastAPI、SQLite、Python 3.11、React 18、Vite、pytest、Node.js 内置测试。

## 1. 全局约束

- 只展示 `open`、`in_progress`、`waiting` Project，并要求最近一次可追溯 Event 与当前时间至少跨一个自然日。
- 每个 Project 只生成一句话摘要；状态词限定为“已完成、进行中、尚未开始、被……阻塞”，不生成“待确认”区块。
- 当前 Session 关联 Project 优先；其余按 `in_progress`、`waiting`、`open`，同状态最近更新优先。
- 默认展示 3 个 Project，其余折叠；Project 名称进入详情，“回到 Session”进入最近相关对话。
- 只读访问不得创建 Event、Radar 或修改 Project / Item。
- 当前工作区已有未提交改动，本计划只修改明确列出的文件，不创建提交，不覆盖无关差异。

## 3. 完成标准

- 隔日返回时能看到未完成 Project 的一句话恢复简报；同日连续工作不展示。
- 多 Project 默认 3 条、可以展开全部，并按已确认顺序稳定排列。
- Project 详情和来源 Session 两条路径都可达，返回后不丢失原对话位置。
- 简报读取不会修改 Event、Item、Project 或生成 Radar。
- 后端目标回归、前端纯函数测试、生产构建和 Ruff 全部通过。

当前实现已满足自动化验收；剩余工作是浏览器中验证跨自然日数据、收起/展开、Project 详情和来源 Session 两条导航路径。
