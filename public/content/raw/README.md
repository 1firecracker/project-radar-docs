## 1. 项目简介

Project Radar 是 AgentV3 主聊天页中的主动推进能力。它在任务边界理解项目进展，发现用户尚未明确提出、但当前最值得推进的一件事，并在正确会话中以低打扰方式提示用户继续行动。

核心原则：

> Radar 跨会话理解项目，但只在正确会话中行动。

## 4. 文档管理规则

### 4.1 唯一权威来源

- 产品定位、用户问题和产品原则只在《产品概要》中维护。
- V0 范围、交互和验收口径只在《零版产品需求》中维护。
- 数据结构、接口和执行流程只在《技术设计》中维护。
- 任务进度只在本文件和《实施计划》中维护。
- 测试用例和演示脚本只在《测试与演示》中维护。
- 其他文档引用权威文档，不复制完整内容。

### 4.2 文档状态

| 状态 | 含义 |
|---|---|
| 草稿 | 内容正在形成，不应作为最终承诺 |
| 评审中 | 内容已完整，等待确认 |
| 已批准 | 已成为当前实现依据 |
| 维护中 | 会随项目进度持续更新 |
| 已归档 | 仅保留历史背景，不再作为当前依据 |

### 4.3 更新要求

- 每次变更必须更新“最近更新”日期。
- 产品范围变化时，同时检查需求、技术设计、实施计划和测试用例。
- 关键方案发生变化时，新增或更新一份决策记录。
- 完成工程任务后，先更新《实施计划》，再更新本文件的当前进度。

## 5. 术语

| 术语 | 含义 |
|---|---|
| Project Radar | 面向项目推进的主动建议能力 |
| Proactive Agent | 在合适时机主动发现并推进工作的 Agent 能力 |
| Radar Item | 一条可解释、可执行、可反馈的推进建议 |
| Action Bar | 输入框上方的低打扰推进提示 |
| Post-run Scan | V0 唯一的 Radar 生成触发器；主 Agent 完成本轮回复或任务后执行扫描 |
| Re-entry | V0 的展示边界；用户重新进入时只读取、校验并展示已有建议，不生成新建议 |
| Session End | V1 的预整理边界；V0 不在此生成 Radar |
| Context Bundle | 点击推进时，由后端重新构造的任务上下文包 |
