## 1. Re-entry 与未来触发器

- V0 的 Re-entry 只读取 Project / Item / Event 概览和已有 Radar，不调用 Organizer 或 Radar Analyzer。
- 前端收到正常 `end_stream` 后立即查询一次，并请求最新真实 QA 的 Radar 分析状态；只要该 QA 仍在 Organizer / Analyzer 链路中，就按有界退避继续查询，最长约 63 秒。出现可展示 Radar，或 `radar_decisions` 已落定为静默、失效、非法或失败结果时立即停止。Re-entry 仍只查询一次。
- 轮询窗口是界面等待上限，不是 Post-run 触发器；Post-run 仍由每轮完整 QA 的 `end_stream` 唯一触发，查询接口不运行 Organizer 或 Analyzer。
- V0.2 的 Resume Brief 由 Re-entry 触发，只读取已有状态并生成恢复视图；当前实现使用确定性摘要，不创建 Event，也不修改 Item / Project。
- Session End、时间扫描、停滞检测、夜间重组、Item 自动合并拆分和全局重新归类属于 V1。
- V0 演示配置关闭 next questions，避免多个后台任务并发使用同一个 LLM；后续可以提供独立 `radar_llm`。

## 2. Resume Brief（V0.2）

Resume Brief 是 Re-entry 阶段的只读状态恢复视图，不是 Post-run 的另一个分支，也不触发 Organizer、Radar 或任何写操作。

### 2.1 触发与数据边界

- 用户登录后进入 Web 页面时调用 `GET /api/resume-brief`；如果已有当前 Session，附带 `current_session_id` 供排序使用。
- 服务端在一个只读事务中读取当前用户的活跃 Project、非归档 Item 和已分类 Event。
- 只有最近一个可追溯 Event 的上海自然日早于今天，且 Project 仍为 `open`、`in_progress` 或 `waiting` 时，Project 才进入简报。
- 摘要由 Item 当前状态确定性拼接为一句话；不从页面访问创建 Event，不通过模型补写事实。
- `brief_version` 由 Project 版本、Item 版本和来源 Event 组成，用于前端收起状态隔离和变更识别。

### 2.2 展示与导航

```mermaid
sequenceDiagram
  participant Web as Web 页面
  participant API as Resume Brief API
  participant Store as Project / Item / Event
  participant Modal as Project 详情
  participant Session as 来源 Session

  Web->>API: Re-entry 查询（可带当前 Session）
  API->>Store: 只读快照
  Store-->>API: 活跃项目、事项状态、最近 Event
  API-->>Web: 最多项目列表 + 一句话摘要
  Web->>Modal: 点击项目名 / 查看项目
  Modal-->>Web: 关闭后回到原对话
  Web->>Session: 点击回到 Session
  Session-->>Web: 定位来源对话，不自动发送消息
```

- 默认展示 3 个 Project；当前 Session 关联项目优先，再按 `in_progress`、`waiting`、`open` 和最近活动时间排序。
- 项目名称和“查看项目”共用现有 Project Radar 详情弹窗；来源 Session 入口复用已有会话选择逻辑。
- Web 展示名为 `Recent Work`，内部服务与接口仍使用 Resume Brief，避免把产品文案耦合到数据模型。
- “收起”后仍保留轻量入口；“× 关闭”通过 `sessionStorage` 在本次浏览器访问中隐藏整个组件；用户发送新 Query 时自动收起。这些操作都不改变持久状态。

### 2.3 接口与实现位置

```text
GET /api/resume-brief?current_session_id=<session-id>
```

返回 `brief_version`、`generated_at` 和 `projects[]`；每个项目包含 `summary`、`source_session_id`、`source_event_id`、`session_ids`，不返回内部提示词或未持久化推断。实现位于 `backend/resume_service.py`、`backend/database.py`、`backend/server.py` 和 `frontend/src/components/ResumeBriefPanel.jsx`。

## 3. 前端接入

- `frontend/src/App.jsx`：维护 Overview、Radar、Resume Brief、侧栏 Tab 和弹窗状态，接入 end-stream 状态感知有界轮询和现有发送流程。
- `frontend/src/resumeApi.js`、`frontend/src/components/resumeBriefState.js`：封装恢复简报读取、排序、折叠状态、当前访问关闭状态和版本隔离。
- `frontend/src/components/ResumeBriefPanel.jsx`：以 Recent Work 为 Web 展示名，展示多项目一句话摘要，提供 Project 详情、来源 Session 双入口，以及“收起”和“× 关闭”。
- 新增 `WorkspaceSidePanel` 与 `RadarPanel`：在现有右侧工作区提供 `Todolist / Radar` 双 Tab；Radar 展示单条建议并提供双栏弹窗入口。
- 新增 `ProjectRadarModal`：居中双栏展示 Project、Item、Non Project 和待分类。
- 新增 `RadarActionBar`：展示单条主动建议，并在同文件内提供原因、证据、反馈和动作详情。
- 新增 `radarApi.js`：封装 Overview、Event、纠正、Radar 和反馈接口。
- 新增 `radarExecution.js`：校验接受结果，构造同 Session 执行参数并合并重复点击。
